# aube bug reproductions

Minimal test cases for bugs found migrating a large monorepo from pnpm to aube.
Each directory is self-contained: `cd` into it and run the commands in its README.

GitHub Actions workflows run each repro on push, comparing aube's behavior against pnpm.

## Fixed in beta.10

1. **`alias-override-conflict/`** — *Fixed in beta.10.* `npm:` alias replaced by an
   unrelated `pnpm.overrides` entry.

2. **`overrides-not-read-from-workspace/`** — *Fixed in beta.10.* `overrides` in
   `pnpm-workspace.yaml` ignored by `--frozen-lockfile`.

3. **`pnpm-lockfile-host-only-platforms/`** — *Fixed in beta.10.* `pnpm-lock.yaml`
   generated by aube only contained optional deps for the host platform.

## Fixed in beta.12

4. **`filter-skips-root-deps/`** — *Fixed in beta.12.* `--filter` skipped root
   workspace `devDependencies`.

## Open bugs (as of beta.12)

5. **`frozen-lockfile-catalog-override/`** — `--frozen-lockfile` rejects `catalog:`
   references in overrides. pnpm resolves `catalog:` to the catalog value before
   comparing against the lockfile. aube compares the raw `catalog:` string, so it
   always reports a mismatch. Workaround: replace `catalog:` in overrides with
   literal values.
